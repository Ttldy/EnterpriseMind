# EnterpriseMind 阶段 0：零上下文完整代码教程

## 0. 学完这一阶段，你会得到什么

完成本教程后，你会亲手搭建出下面这条最小但完整的 Agent 主链：

```text
HTTP 请求
→ FastAPI /api/v1/chat
→ trace_id 中间件
→ AgentOrchestrator
→ RuleRouter
→ 选择 HR / IT / Finance / DataAnalyst Agent Prompt
→ ModelProvider
→ HTTP 响应
```

项目此时还没有数据库、登录、RAG、Ollama 和前端。这是故意的。

阶段 0 的目标不是一次做完整系统，而是先建立一条可运行、可测试、可替换模型的主干。后续 PostgreSQL、Qdrant、权限和 Ollama 都会沿着这条主干接入。

最终可以运行：

```powershell
Invoke-RestMethod `
  -Method Post `
  -Uri "http://127.0.0.1:8000/api/v1/chat" `
  -ContentType "application/json" `
  -Body '{"message":"公司 VPN 无法连接"}'
```

预期返回结构：

```json
{
  "answer": "【演示模型】已由 it Agent 处理：公司 VPN 无法连接",
  "agent": "it",
  "intent": "knowledge_query",
  "model": "demo-model",
  "sensitivity": "internal",
  "trace_id": "一串32位十六进制字符串"
}
```

## 1. 本阶段的技术选择

| 技术 | 用途 | 为什么现在使用 |
|---|---|---|
| Python 3.12 | 后端语言 | 类型系统、异步生态和 AI 工程库成熟 |
| FastAPI | HTTP API | 基于类型生成校验与 OpenAPI，适合 Agent 服务 |
| Pydantic 2 | 请求和响应模型 | 在 HTTP 边界校验输入输出 |
| dataclass | 业务层数据结构 | 不让业务层依赖 FastAPI/Pydantic |
| Protocol | 模型 Provider 接口 | 不绑定具体模型 SDK，方便 Fake 和替换 |
| pytest | 自动测试 | 使用 RED → GREEN 建立可回归行为 |
| Ruff | 代码质量检查 | 快速统一代码规范并发现常见错误 |
| MyPy | 静态类型检查 | 在运行前发现接口不匹配 |
| httpx | 外部模型 HTTP 调用 | 同时支持异步请求和测试场景 |

### 为什么不用 LangChain 或 LangGraph

阶段 0 的学习目标是理解 Agent 最核心的调用链和边界。如果直接使用大型 Agent 框架，你容易只记住框架 API，而说不清：

- 路由结果是什么；
- 编排器负责什么；
- Prompt 在哪一层；
- 模型如何被替换；
- HTTP 模型与业务模型为什么分开。

因此阶段 0 使用原生 Python 和 FastAPI。等你理解底层边界后，再学习 LangGraph 会更稳。

现代 Python 支持没有 `__init__.py` 的命名空间包，但学习项目中显式保留它更清晰：

- 明确告诉 Python 和开发者“这里是一个包”；

- 后续可在包初始化时导出公共类型；
- IDE 的跳转和类型检查更稳定。

### 技术理解卡：应用工厂

`create_app()` 是应用工厂，而模块级 `app = create_app()` 是生产启动入口。

两者并存的原因：

- Uvicorn 可以导入 `app.main:app`；
- 测试可以调用 `create_app()` 获得全新应用；
- 后续测试可以注入 Fake Provider；
- 不同环境可以传入不同配置；
- 减少全局单例污染测试。

## 7. 任务二：定义 Agent 业务契约

### 7.1 创建业务类型

创建：

```text
backend/app/agents/contracts.py
```

完整内容：

```python
from dataclasses import dataclass
from enum import StrEnum


class AgentType(StrEnum):
    HR = "hr"
    IT = "it"
    FINANCE = "finance"
    DATA_ANALYST = "data_analyst"
    CLARIFICATION = "clarification"


class IntentType(StrEnum):
    KNOWLEDGE_QUERY = "knowledge_query"
    DATA_QUERY = "data_query"
    UNKNOWN = "unknown"


class Sensitivity(StrEnum):
    PUBLIC = "public"
    INTERNAL = "internal"
    SENSITIVE = "sensitive"


@dataclass(frozen=True)
class RouteResult:
    agent: AgentType
    intent: IntentType
    requires_sql: bool
    sensitivity: Sensitivity
    confidence: float


@dataclass(frozen=True)
class OrchestratorResult:
    answer: str
    agent: AgentType
    intent: IntentType
    model: str
    sensitivity: Sensitivity
```

### 为什么用 `StrEnum`

`StrEnum` 的成员既是枚举，又可以自然序列化为字符串：

```python
AgentType.IT.value == "it"
```

相比到处使用裸字符串，它可以在 MyPy 和 IDE 中发现拼写错误。

### 为什么业务层使用 `dataclass`

`ChatRequest` 和 `ChatResponse` 是 HTTP 边界模型，应该使用 Pydantic。

`RouteResult` 和 `OrchestratorResult` 是内部业务模型，使用标准库 `dataclass`：

- 不依赖 FastAPI；
- 创建成本低；
- 接口明确；
- 以后可在命令行、任务队列或测试中复用。

`frozen=True` 表示对象创建后不可修改，可避免某一层无意改变路由决策。

### 8.2 实现路由器

创建：

```text
backend/app/agents/router.py
```

完整内容：

```python
from app.agents.contracts import (
    AgentType,
    IntentType,
    RouteResult,
    Sensitivity,
)


class RuleRouter:
    _domain_rules: dict[AgentType, tuple[str, ...]] = {
        AgentType.HR: (
            "年假",
            "请假",
            "考勤",
            "福利",
            "入职",
            "离职",
        ),
        AgentType.IT: (
            "vpn",
            "登录",
            "报错",
            "设备",
            "网络",
            "密码",
        ),
        AgentType.FINANCE: (
            "报销",
            "发票",
            "差旅",
            "付款",
            "预算",
        ),
    }

    _statistics_words: tuple[str, ...] = (
        "统计",
        "趋势",
        "汇总",
        "各部门",
        "本月多少",
    )

    def route(self, message: str) -> RouteResult:
        normalized = message.strip().lower()

        if self._requires_data_query(normalized):
            return RouteResult(
                agent=AgentType.DATA_ANALYST,
                intent=IntentType.DATA_QUERY,
                requires_sql=True,
                sensitivity=Sensitivity.SENSITIVE,
                confidence=0.95,
            )

        for agent, keywords in self._domain_rules.items():
            if any(keyword in normalized for keyword in keywords):
                return RouteResult(
                    agent=agent,
                    intent=IntentType.KNOWLEDGE_QUERY,
                    requires_sql=False,
                    sensitivity=Sensitivity.INTERNAL,
                    confidence=0.90,
                )

        return RouteResult(
            agent=AgentType.CLARIFICATION,
            intent=IntentType.UNKNOWN,
            requires_sql=False,
            sensitivity=Sensitivity.INTERNAL,
            confidence=0.20,
        )

    def _requires_data_query(self, message: str) -> bool:
        return any(word in message for word in self._statistics_words)
```

### 技术理解卡：为什么“规则优先”

路由属于控制平面。能用确定性规则判断的请求，不应该先付出 LLM 的延迟、成本和不确定性。

规则优先的优点：

- 本地执行，通常是微秒级；
- 同一输入必然得到同一结果；
- 安全规则容易测试；
- 不依赖网络；
- 便于解释路由原因。

缺点是覆盖范围有限。后续阶段会增加 LLM 结构化分类作为兜底，而不是替换规则。

当前 `_requires_data_query()` 只是在搭建骨架，不代表生产级意图识别能力。

## 9. 任务四：建立模型 Provider 抽象

### 9.1 创建 Provider 契约

创建：

```text
backend/app/model_gateway/contracts.py
```

完整内容：

```python
from dataclasses import dataclass
from typing import Protocol


@dataclass(frozen=True)
class ModelRequest:
    system_prompt: str
    user_message: str


@dataclass(frozen=True)
class ModelResponse:
    text: str
    model: str


class ModelProvider(Protocol):
    async def generate(self, request: ModelRequest) -> ModelResponse:
        ...
```

### Protocol 的意义

`Protocol` 是结构化子类型，也叫静态鸭子类型。

一个类不需要显式继承 `ModelProvider`，只要拥有签名兼容的异步 `generate()` 方法，MyPy 就认为它满足接口。

因此以下对象都可以注入编排器：

- `DemoProvider`；
- `ExternalModelProvider`；
- 后续的 `OllamaProvider`；
- 测试中的 `FakeProvider`。

这就是依赖倒置：业务层依赖抽象，不依赖具体 SDK。

### 9.3 创建真实外部 HTTP Provider

创建：

```text
backend/app/model_gateway/external.py
```

完整内容：

```python
from typing import Any

import httpx

from app.model_gateway.contracts import ModelRequest, ModelResponse


class ExternalModelProvider:
    def __init__(
        self,
        base_url: str,
        api_key: str,
        model: str,
        timeout_seconds: float = 30.0,
    ) -> None:
        self._base_url = base_url.rstrip("/")
        self._api_key = api_key
        self._model = model
        self._timeout_seconds = timeout_seconds

    async def generate(self, request: ModelRequest) -> ModelResponse:
        headers = {
            "Authorization": f"Bearer {self._api_key}",
            "Content-Type": "application/json",
        }
        payload: dict[str, Any] = {
            "model": self._model,
            "messages": [
                {
                    "role": "system",
                    "content": request.system_prompt,
                },
                {
                    "role": "user",
                    "content": request.user_message,
                },
            ],
            "temperature": 0.1,
        }

        async with httpx.AsyncClient(timeout=self._timeout_seconds) as client:
            response = await client.post(
                f"{self._base_url}/chat/completions",
                headers=headers,
                json=payload,
            )
            response.raise_for_status()
            data = response.json()

        text = str(data["choices"][0]["message"]["content"])
        return ModelResponse(
            text=text,
            model=self._model,
        )
```

这段实现采用 OpenAI 兼容的 `/chat/completions` 结构。阶段 0 不默认启用它，以免 API 配置问题干扰主链学习。

import pytest

from app.agents.contracts import AgentType
from app.agents.orchestrator import AgentOrchestrator
from app.agents.router import RuleRouter
from app.model_gateway.contracts import ModelRequest, ModelResponse


class FakeProvider:
    def __init__(self) -> None:

    async def generate(self, request: ModelRequest) -> ModelResponse:
        self.last_request = request
        return ModelResponse(
            text=f"fake answer: {request.user_message}",
            model="fake-model",
        )


@pytest.mark.asyncio
async def test_orchestrator_selects_hr_prompt() -> None:
    provider = FakeProvider()
    orchestrator = AgentOrchestrator(
        router=RuleRouter(),
        provider=provider,
    )

    result = await orchestrator.run("年假有几天？")

    assert result.agent is AgentType.HR

```python
from app.agents.contracts import (
    AgentType,
    OrchestratorResult,
)
from app.agents.domain_agents import PROMPTS
from app.agents.router import RuleRouter
from app.model_gateway.contracts import (
    ModelProvider,
    ModelRequest,
)


class AgentOrchestrator:
    def __init__(
        self,
        router: RuleRouter,
        provider: ModelProvider,
    ) -> None:
        self._router = router
        self._provider = provider

    async def run(self, message: str) -> OrchestratorResult:
        route = self._router.route(message)

        if route.agent is AgentType.CLARIFICATION:
            return OrchestratorResult(
                answer="请补充问题所属领域和具体需求。",
                agent=route.agent,
                intent=route.intent,
                model="none",
                sensitivity=route.sensitivity,
            )

        prompt = PROMPTS[route.agent]
        model_response = await self._provider.generate(
            ModelRequest(
                system_prompt=prompt,
                user_message=message,
            )
        )

        return OrchestratorResult(
            answer=model_response.text,
            agent=route.agent,
            intent=route.intent,
            model=model_response.model,
            sensitivity=route.sensitivity,
        )
```

## 11. 任务六：trace_id 中间件

创建：

```text
backend/app/shared/trace.py
```

完整内容：

```python
from collections.abc import Awaitable, Callable
from uuid import uuid4

from fastapi import Request, Response
from starlette.middleware.base import BaseHTTPMiddleware


class TraceIdMiddleware(BaseHTTPMiddleware):
    async def dispatch(
        self,
        request: Request,
        call_next: Callable[[Request], Awaitable[Response]],
    ) -> Response:
        trace_id = uuid4().hex
        request.state.trace_id = trace_id

        response = await call_next(request)
        response.headers["X-Trace-ID"] = trace_id
        return response
```

### 中间件执行顺序

一次请求进入时：

```text
客户端
→ TraceIdMiddleware 生成 trace_id
→ 路由函数读取 request.state.trace_id
→ 路由返回 Response
→ 中间件写入 X-Trace-ID 响应头
→ 客户端
```

阶段 0 只生成 trace ID。后续阶段会用同一个 ID 串联认证、路由、检索、模型和 SQL。

### 12.2 创建 API 模型与路由

创建：

```text
backend/app/api/chat.py
```

完整内容：

```python
from fastapi import APIRouter, Request
from pydantic import BaseModel, Field

router = APIRouter(tags=["chat"])


class ChatRequest(BaseModel):
    message: str = Field(
        min_length=1,
        max_length=4000,
    )


class ChatResponse(BaseModel):
    answer: str
    agent: str
    intent: str
    model: str
    sensitivity: str
    trace_id: str


@router.post("/chat", response_model=ChatResponse)
async def chat(
    body: ChatRequest,
    request: Request,
) -> ChatResponse:
    result = await request.app.state.orchestrator.run(body.message)

    return ChatResponse(
        answer=result.answer,
        agent=result.agent.value,
        intent=result.intent.value,
        model=result.model,
        sensitivity=result.sensitivity.value,
        trace_id=request.state.trace_id,
    )
```

### 为什么 HTTP 模型与业务模型分开

`ChatRequest`/`ChatResponse` 属于传输层，负责：

- JSON 校验；
- 字段长度；
- OpenAPI 文档；
- HTTP 序列化。

`RouteResult`/`OrchestratorResult` 属于业务层，负责：

- 领域语义；
- 层间协作；
- 不依赖 HTTP 的业务结果。

如果混在一起，未来 RQ Worker、命令行或 WebSocket 想调用编排器时，就会被迫依赖 FastAPI 的模型。

## 21. 面试时如何讲阶段 0

### 30 秒版本

> 我先没有直接上 RAG 和数据库，而是用 FastAPI 建了一条最小 Agent 主链。HTTP 层只处理请求响应，Orchestrator 负责路由和调用 Agent，模型能力通过 Protocol 抽象成 Provider，因此测试可以注入 Fake，运行时可以切换 Demo、外部模型或后续的 Ollama。每个请求由中间件生成 trace_id，为后续权限、检索和 SQL 链路追踪打基础。

### 面试官：为什么不直接把所有代码写在 `/chat` 中

回答：

> `/chat` 是传输层。如果把路由、Prompt、模型 SDK 和业务逻辑都放进去，后续接入 SSE、任务队列、测试或其他入口时会重复代码。我把 HTTP 模型、业务契约、路由、编排和 Provider 分层，使每层可以独立测试和替换。

### 面试官：为什么用 Protocol

回答：

> AgentOrchestrator 不应该依赖某个具体模型 SDK，只需要一个具有异步 generate 方法的对象。Protocol 提供静态鸭子类型，不要求显式继承，FakeProvider、ExternalProvider 和 OllamaProvider 都能满足同一个契约，从而实现依赖倒置和可测试性。

### 面试官：为什么规则优先，LLM 兜底

回答：

> 明确的高频领域问题可以用确定性规则低延迟处理，而且容易测试和解释。规则无法判断时再使用 LLM 结构化分类，可以降低成本和不确定性。安全相关决策也不应该只依赖 LLM。

### 面试官：DemoProvider 有什么意义

回答：

> 它让系统在没有 API Key 和网络的情况下仍能验证完整编排链路，避免把基础架构问题和模型服务问题混在一起。自动测试使用 FakeProvider，开发演示默认使用 DemoProvider，真实环境再注入外部或本地 Provider。

### 面试官：阶段 0 哪些地方还不是企业级

回答：

> 当前路由只使用简单关键词；没有认证、权限、RAG、持久化、模型安全路由和真实 SQL。阶段 0 的价值是建立可扩展、可测试的主链边界，后续企业能力可以沿这些接口接入，而不是推倒重写。

## 22. 你应该真正理解的调用链

当用户发送：

```json
{"message":"公司 VPN 无法连接"}
```

代码执行顺序：

```text
1. Uvicorn 接收 HTTP 请求
2. FastAPI 构造 Request
3. TraceIdMiddleware 生成 trace_id
4. FastAPI 根据 ChatRequest 校验 JSON
5. chat() 从 app.state 取得 AgentOrchestrator
6. AgentOrchestrator.run() 调用 RuleRouter.route()
7. RuleRouter 命中 IT 关键词
8. Orchestrator 从 PROMPTS 取得 IT Prompt
9. Orchestrator 构造 ModelRequest
10. DemoProvider.generate() 返回 ModelResponse
11. Orchestrator 构造 OrchestratorResult
12. chat() 构造 ChatResponse
13. Pydantic 序列化为 JSON
14. TraceIdMiddleware 写入 X-Trace-ID 响应头
15. Uvicorn 返回 HTTP 响应
```

你能够不看代码讲出这 15 步，阶段 0 才算真正掌握。
